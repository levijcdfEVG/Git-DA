Introducción a GIT en Windows

Git es una de las herramientas más utilizadas para el control de versiones, permitiendo a los desarrolladores gestionar y rastrear los cambios en el código de sus proyectos. Para instalar Git en Windows, simplemente descarga el instalador desde la página oficial de Git y sigue las instrucciones de instalación. Una vez completada la instalación, podrás utilizar Git desde la línea de comandos (Git Bash) o integrarlo con editores de código como Visual Studio Code. Este proceso es crucial para quienes trabajan en equipo, ya que Git permite coordinar y colaborar eficientemente con otros desarrolladores, manteniendo un historial detallado de los cambios.

Configuración Inicial de GIT

Después de instalar Git, el siguiente paso es configurarlo con tu nombre y correo electrónico, los cuales serán utilizados para identificarte en cada commit. Esta configuración solo es necesaria una vez, pero puede actualizarse si lo necesitas en el futuro. Para realizar esta configuración inicial, utiliza los siguientes comandos desde la terminal de Git Bash:

  • git config --global user.name "Tu Nombre"
  • git config --global user.email "tuemail@ejemplo.com"

Además, puedes verificar tu configuración en cualquier momento utilizando git config --list, que mostrará todos los parámetros configurados a nivel global, como el nombre de usuario y el correo electrónico.

Creación de un Repositorio

Una vez configurado Git, puedes crear un nuevo repositorio en cualquier carpeta de tu proyecto con el comando git init. Esto inicializa un repositorio local en esa carpeta, preparando el entorno para empezar a rastrear cambios. Luego, usa git status para ver el estado de tus archivos. Los archivos nuevos aparecerán como "untracked", lo que significa que aún no están siendo rastreados por Git.

Añadir archivos

Para agregar archivos al área de preparación (staging), usa el comando git add [archivo] o git add . si deseas incluir todos los archivos del proyecto. Esto prepara los archivos para ser confirmados (commit). Una vez que hayas agregado los archivos, realiza un commit para guardar los cambios en el repositorio local usando git commit -m "Mensaje del commit". Ahora tus cambios estarán almacenados de manera segura en el historial del repositorio.

Después de confirmar los cambios con `git commit`, es posible que desees subir estos cambios a un repositorio remoto, como GitHub o GitLab, para compartirlos con otros colaboradores o simplemente para mantener una copia segura en la nube.

Para hacerlo, primero debes vincular tu repositorio local con uno remoto mediante el comando `git remote add origin [URL]`, donde la URL es la dirección de tu repositorio en la plataforma remota. Una vez configurado, puedes enviar los cambios con `git push -u origin main`, lo que subirá tus commits a la rama principal del repositorio remoto.

A partir de ahí, cualquier colaborador con acceso podrá clonar el repositorio y trabajar en el proyecto de manera colaborativa.
